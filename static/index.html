Документація до АРІ

Робота з користувачем:

GET - перегляд

  • Щоб отримати усіх користувачів зверніться за адресою:...api/users
  • В результаті до вас прийде масив об'єктів (користувачів).
  • Для отримання одного користувача, потрібно передати id.
  • Тому робимо наступним чином:...api/users/:id - де id це потрібний користувач.

POST - додавання

  • Щоб створити нового користувача зверніться за адресою:...api/users
  • Важливий нюанс, важливо передати в тілі запиту наступні дані:
  • first_name, last_name, login, password, email, phone_number - без них нічого не працюватиме.
  • Що робити якщо користувач не створюється?
  • Читаємо відповідь від АРІ,скоріш за все такий логін та пароль вже існує

PUT - оновлення

  • Щоб оновити користувача зверніться за адресою:...api/users
  • В тіло запиту обов'язково передайте id потрібного користувача та базове тіло запиту:
  • first_name, last_name, login, password, email, phone_number

DELETE - видалення

  • Для видалення користувача зверніться за адресою:...api/users/:id
  • Важливо, якщо будете тестувати видалення, не забудьте перед тим постворювати користувачів.
